# 06 7天实战任务清单（每天照做）

定位：给零基础用户的执行版清单。每天 30-90 分钟，7 天把 OpenClaw 从“能跑”升级到“真能用”。

使用方法：

- 每天只做当日任务，不跳级。
- 每天结束时，必须完成“验收标准”。
- 验收不通过，第二天先补齐再继续。

---

## Day 1：跑通最小闭环

目标：完成安装 + onboarding + 第一条成功对话。

任务：

1. 安装 OpenClaw。
2. 跑 onboarding（建议 QuickStart）。
3. 完成一次本地对话测试。
4. 记录当前版本和运行状态。

可用命令：

```bash
npm install -g openclaw@latest
openclaw onboard --install-daemon
openclaw status
openclaw gateway status
openclaw agent --message "你好，做个3行自我介绍" --thinking low
```

验收标准：

- `openclaw status` 显示核心服务正常。
- 能拿到至少 1 条正确回复。

---

## Day 2：先上安全基线

目标：从第一天起就把风险降到可控。

任务：

1. 打开配置文件（通常是 `~/.openclaw/openclaw.json`）。
2. 配置 allowlist，只允许你自己访问。
3. 群聊设置为必须 @ 才触发（如果你会在群里用）。
4. 跑一次健康检查。

建议检查点：

- 是否限制了可访问账号。
- 是否避免了“群里任何人都能直接调用”。

验收标准：

- 白名单生效，非授权账号不能调用。
- 群聊未 @ 时不触发。

---

## Day 3：接入一个消息渠道（建议 Telegram）

目标：从手机可随时调用助手。

任务：

1. 接入一个渠道（先只接 1 个）。
2. 在手机端完成 `/start` 或首次握手。
3. 连续发送 3 条不同类型请求：
   - 问答
   - 总结
   - 任务提醒

可用命令：

```bash
openclaw channels login
openclaw status
```

验收标准：

- 手机端可稳定收发。
- 3 种请求都能正常完成。

---

## Day 4：装 2-3 个高频技能并做首轮验证

目标：把“聊天助手”升级成“任务助手”。

任务：

1. 按你的高频场景选 2-3 个技能（例如：搜索、总结、日程/邮件）。
2. 每个技能至少跑 1 次真实任务。
3. 记录：这个技能解决了什么、有什么限制。

建议：

- 先装高频、低风险技能。
- 涉及外部账号授权的技能，优先用测试账号。

验收标准：

- 至少 2 个技能可稳定使用。
- 你写下了每个技能的适用边界。

---

## Day 5：做 3 条自动化（提醒/周期任务）

目标：让 OpenClaw 主动帮你做事。

任务：

1. 配置 1 条一次性提醒（例如：30 分钟后提醒）。
2. 配置 1 条每日任务（例如：晚间日报总结）。
3. 配置 1 条每周任务（例如：周复盘提醒）。

建议模板：

- 一次性：`30 分钟后提醒我开会前准备资料`
- 每日：`每天 21:30 提醒我做当天复盘`
- 每周：`每周日 20:00 提醒我做下周计划`

验收标准：

- 3 条任务都创建成功。
- 至少有 1 条按预期触发。

---

## Day 6：做一个完整实战流

目标：跑通“输入 -> 处理 -> 输出 -> 跟进”的完整链路。

任务（示例流）：

1. 给出一个真实问题（如：下周工作安排）。
2. 让 OpenClaw输出结构化结果（优先级 + 时间块）。
3. 将结果转成提醒或待办。
4. 当天结束前复盘：结果是否可执行。

你也可以选自己的流：

- 邮件清理流
- 学习资料总结流
- 项目周报流

验收标准：

- 完成 1 个可复用流程。
- 该流程至少帮你节省一次重复劳动。

---

## Day 7：固化与交付（长期可用）

目标：从“试用”升级为“长期稳定使用”。

任务：

1. 整理一份你的个人使用 SOP（10-20 行即可）。
2. 固化 5 条你常用提示词模板。
3. 做一次备份策略确认（配置、工作目录、关键文档）。
4. 确认更新策略（稳定更新频率 + 回滚方案）。

SOP 最小模板：

- 我每天什么时候用
- 我固定触发的 3 个任务
- 出问题先检查哪 3 个地方

验收标准：

- 你有一份自己的 SOP。
- 知道“怎么升级、怎么回滚、怎么恢复”。

---

## 7天完成后的达成标志

- 能稳定对话（本地 + 手机）。
- 有明确安全边界（allowlist/群聊触发策略）。
- 至少 2 个技能可稳定干活。
- 至少 3 条自动化任务在跑。
- 有个人 SOP，可长期维护。
